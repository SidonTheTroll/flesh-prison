## Introduction  
- The internet is a global network of networks, developed by ARPA in 1970.  
- It is used to transmit information.  
- Connections include phone lines, DSL, and wireless.  
- Its operation is maintained by various organizations and users.  

# Evolution  

| Year | Glimpse | Development |
|:-:|:-|:-|
| 1960s | ARPANET connects research centers | 1969: 15 nodes, 23 hosts established |
| 1970s | Email, packet switching, protocols | Packet switching enabled long-distance communication |
| 1980s | WWW, browsers, file sharing | World Wide Web invented |
| 1990s | Domains, servers, security | Internet commercialized; ARPANET ended |
| 2000s | Web 2.0 | Blogs, wikis, social media |
| 2010s | Web 3.0 | Semantic web, data sharing |

# Intranet  
- A private network used within an organization for communication and data sharing.  

## Uses  
1. Simplifies routine tasks.  
2. Centralizes company data.  
3. Improves collaboration.  
4. Enhances internal communication.  
5. Provides quick access to policies and updates.  

## Internet vs Intranet
| Parameter | Internet | Intranet | 
|-|-|-|
| Usage | Public | Private | 
| User Types | Any user having dial up internet access line. | Organizational employees and internal company departments. |
| Usage | Access all kind of information. | Internal employee information, telephone directories, etc. | 
| Security | Low security. Configured under 0 security level in firewall. | High security. Configured under 100 security level in firewall. | 
| Regulated by | Internet Architecture Board (IAB): Oversees the technical and engineering development of IETF and IRTF. Internet Corporation for Assigned Names and Numbers (ICANN). | It is regulated by an organization. | 
| Coverage | Wide Area | Within an organization | 
| Access | Large number of users | Limited number of users | 
| System failure | Unpredictable | System availability is high since system is monitored by authority | 

## Internet Service Provider (ISP)  
- An organization that provides internet access and related services. 

- Factors to consider when choosing an ISP: 
    - **Bandwidth:** data transfer speed. 
    - **Availability:** network uptime and performance. 
    - **Cost:** pricing of plans and services. 
    - **Network security:** protection of user data and privacy. 
    - **Customer service:** quality of support. 
    - **Location & speed:** local providers often offer better speed and support.
    
# Getting Connected  

## Internet Connections  
- Types:  
    1. Dial-up  
    2. DSL  
    3. Cable  
    4. Wireless  
    5. Satellite  
    6. Cellular  

### 1. Dial-up  
- Cheap but slow  
- Uses phone line via modem  
- Converts analog to digital  
- Line quality affects speed  
- No simultaneous phone use  
- Speed: 26–56 Kbps  

### 2. DSL  
- Digital Subscriber Line  
- Always-on connection  
- Separate line for phone  
- No dialing required  
- Uses router  
- Speed: 128 Kbps–8 Mbps  

<br>

- Types: ADSL, SDSL, HDSL, RDSL, VDSL, IDSL  
- Differ in speed and service  

### 3. Cable  
- Uses cable TV lines  
- Higher bandwidth than DSL/dial-up  
- Speed varies (upload/download)  
- Speed: 512 Kbps–20 Mbps  

### 4. Wireless  
- No cables, uses radio signals  
- Always-on connection  
- Accessible anywhere  
- Speed: up to ~20 Mbps  

### 5. Satellite  
- Uses orbiting satellite  
- High latency due to distance  
- Speed: 12 Kbps–2 Mbps  

### 6. Cellular  
- Mobile network access  
- Uses 3G, 4G, 5G  
- 3G ≈ 2 Mbps  
- 4G ≈ 21 Mbps (typical)

# Working of Internet  
- Uses TCP/IP protocols  
- Devices connect via wired/wireless  
- Each device has a unique IP  
- Data split into packets  
- Packets routed via ISP router  
- Router sends packets to destination  

---

# Main Components of Internet  
- **Clients:** user devices (PCs, phones, tablets)  
- **Servers:** provide data/services  
- **Protocols:** rules (IP, TCP/IP, HTTP, HTTPS, FTP)  
- **IP address:** unique device ID (IPv4, IPv6)  
- **Routers:** direct data paths  
- **Browsers:** access and display web content  
- **ISPs:** provide connectivity  
- **Firewalls:** protect networks  
- **Social platforms:** enable communication  

---

### Key Features of Internet  
- **Global access:** available worldwide  
- **Media compatibility:** supports multiple formats  
- **Ease of use:** simple access via browsers  

# Security and Internet  
- Large amounts of data are shared online.  
- Risks: data breaches, malware, ransomware, phishing.  
- Hackers can steal login or financial data.  

- Internet security = methods to protect data and activities.  

## Safety Measures  
1. Use antivirus and anti-malware.  
2. Create strong, unique passwords.  
3. Use VPN or private browsing.  
4. Prefer HTTPS over HTTP.  
5. Keep social accounts private.  
6. Disable autofill.  
7. Turn off GPS when not needed.  
8. Manage cookies carefully.  
9. Log out after use.  
10. Avoid spam and unknown links.  
11. Be careful on public Wi-Fi.  
12. Backup data regularly.

## What is a Browser?  
- Software to access websites.  
- Fetches data from servers and displays web pages.  
- Works like a translator between user and server.  
- Main functions: fetch, display, and enable interaction.  

---

## Types of Web Browser  

### 1. Mobile Browsers  
- Designed for small screens and touch use.  
- Focus on speed and usability.  
- Examples: Safari (iOS), Chrome (Android).  

### 2. Embedded Browsers  
- Built into apps.  
- Limited features compared to full browsers.  

**Examples:**  
- Email apps → view web content  
- Social media apps → open links  
- Gaming consoles → access online features  

### Browser Components  
1. **User Interface:** address bar, tabs, navigation  
2. **Rendering Engine:** displays web pages  
3. **Networking:** fetches data from servers  
4. **JavaScript Engine:** runs scripts  
5. **Security:** encryption and protection  

---

## World Wide Web (WWW)  
- Works on client-server model.  
- Client requests data, server responds.  

### Steps  
1. **DNS lookup:** domain → IP address  
2. **Server request:** browser requests data  
3. **Rendering:** page is displayed  

# Features  

## Search Engines  
- Tools to search content on the web.  
- Work by:  
    - Finding content via keywords  
    - Storing indexed data  
    - Retrieving results  

- Process:  
    - Crawling  
    - Indexing  
    - Query processing  
    - Ranking  

### Crawling  
- Bots (spiders) scan web pages.  
- Follow links and read content.  

### Indexing  
- Stores data in a large database (index).  
- Enables fast searching.  

- Includes:  
    - File/URL, title, size  
    - Full text  
    - Metadata (category, subject)  

### Processing Queries  
- Matches query with indexed data.  
- Uses algorithms to filter results.  

### Ranking  
- Orders results by relevance.  
- Displays most useful first.  

---

## Types of Search Engines  

| Type | Example | Description |
|-|-|-|
| Conventional | Library catalog | Search by title/author |
| Text-based | Google, Bing | Keyword search |
| Voice-based | Google, Bing | Voice queries |
| Multimedia | QBIC, WebSeek | Search by visuals |

### Categories  
- Crawler-based → automated indexing  
- Human-powered → manually curated

## IP Address  
- Unique identifier for each device.  

### IPv4  
- Format: four numbers (0–255)  
- Example: 192.168.1.1  
- ~4 billion addresses  

- Structure:  
    - 4 octets (8 bits each)  
    - Network + host parts  

### IPv6  
- 128-bit address  
- Larger address space  
- Format: hexadecimal groups  

- Example:  
  2001:0db8:85a3:0000:0000:8a2e:0370:7334

## Website  
- Collection of web pages  
- Accessed via URL  

### Types  

#### Static Website  
- Fixed content  
- Uses HTML/CSS/JS  
- Easy to build  
- No database  

#### Dynamic Website  
- Content changes  
- Uses server-side code (PHP, etc.)  
- Uses database  
- More complex

| Static Website | Dynamic Website |
|-|-|

| Simple | Complex |
| No database | Uses database |

# Security 
# Security  

- Security in IT systems refers to protecting data, devices, and networks from unauthorized access, damage, or misuse.  
- It ensures confidentiality, integrity, and availability of information.  

## Common Threats  
- Malware (viruses, worms, ransomware)  
- Phishing attacks  
- Data breaches  
- Unauthorized access (hacking)  

## Security Measures  
- Use strong and unique passwords  
- Install antivirus and anti-malware software  
- Enable firewalls  
- Use secure protocols (HTTPS)  
- Avoid suspicious links and downloads  
- Regularly update software and systems  
- Backup important data  

## Security Tools  
- **Firewalls:** block unauthorized access  
- **Encryption (HTTPS):** protects data in transit  
- **Authentication systems:** verify user identity  
- **Antivirus software:** detects and removes threats  

## Importance  
- Protects personal and organizational data  
- Prevents financial loss and data theft  
- Ensures safe communication over networks
